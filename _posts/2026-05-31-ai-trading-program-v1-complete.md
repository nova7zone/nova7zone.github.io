## KIS 자동매매 프로그램 1차 완성

Claude Code와 Github를 이용해서 한국투자증권(KIS) 자동매매 프로그램을 완성했다.

- 설정 파일 등 편의성을 위한 수정 요청 → 반영 완료
- 최종적으로 완성된 프로그램을 기반으로 **프롬프트 md 파일**을 새로 생성

여기서 만든 프롬프트 md 파일은 이후 Upbit 버전을 만들 때 그대로 재사용했다.

---

## Upbit 버전 추가 개발

KIS 자동매매 프로그램이 한 차례 완성되자, 같은 구조를 가상화폐 거래소인 **Upbit**에도 적용해보기로 했다.

- 한국투자증권 자동매매 프롬프트를 기반으로 일부 내용을 Upbit API에 맞게 수정
- Upbit용 자동매매 프로그램 추가 개발 착수
- Claude에 현재 프로그램의 매매 조건(스크리닝 로직)을 전달하고, 추가로 개선할 사항이 있는지 검토 요청

주식과 코인은 시장 운영 시간, 호가 단위, API 응답 형식이 모두 다르기 때문에 단순 복붙으로는 동작하지 않는다. 다만 "스크리닝 → 매수 → 매도 → 리포트"라는 전체 구조와 설정 파일 기반 동작 방식은 그대로 재사용할 수 있었다.

## 2차 스크리닝 필터 2종 복원

처음 설계 단계에서는 총 8개 필터 중 2개를 제외하고 6개로 시작했는데, 운영해보니 빠진 2개가 필요했다.

- **ATR 변동성 필터**: 변동성이 지나치게 큰 종목을 걸러내는 필터
- **시세 국면 필터**: 시장이 상승/하락/횡보 중 어떤 국면인지에 따라 매수 기준을 조정하는 필터

추가 방식은 다음과 같았다.

1. Claude(웹)에 필터 로직 작성을 요청해 소스 코드를 받음
2. 받은 소스 코드를 Claude Code 프로젝트에 그대로 전달
3. "이 코드를 프로그램에서 사용할 수 있도록 추가해줘"라고 요청해 통합

이렇게 하면 Claude Code가 기존 프로젝트의 모듈 구조(스크리닝 조건 모듈 인터페이스)에 맞춰 새 필터를 등록해준다.

## 설정 변수 통합 문서 작성

설정 가능한 변수들이 여러 YAML 파일에 흩어져 있어서, 어떤 값을 바꾸려면 매번 여러 파일을 열어봐야 했다. 이를 해결하기 위해 다음을 요청했다.

- 임의로 변경이 필요한 변수들을 **하나의 md 파일**에서 한눈에 확인할 수 있도록 정리

이후 커밋 및 PR 확인을 요청한 뒤, 작업 폴더를 구글 클라우드에 업로드하고 그날 작업을 종료했다.

---

## 멀티 PC 작업 환경 준비

다른 컴퓨터에서 이어서 작업할 수 있도록 환경을 준비하는 과정도 함께 진행했다.

1. 다른 컴퓨터에서 작업을 준비
2. Claude **데스크탑 앱**과 **CLI(Claude Code)**의 차이를 비교 검토
3. node.js 설치
4. python 설치
5. 새로운 세션에서 작업을 진행했는데, 세션 이름이 표시되지 않는 문제 발견
6. 수정 작업은 정상적으로 진행되었고, Github 연동 커밋·PR까지 완료
7. 확인 결과, 다른 컴퓨터에서 나눈 대화 내용은 새 세션에서 알 수 없음 — 원격 세션 기능은 있지만 이번 작업에는 필요 없다고 판단
8. VS Code + Claude Code + Github 연동으로 작업 진행
9. VS Code 설치 후 작업 폴더를 열어 이어서 작업

핵심은 "대화 맥락"이 아니라 "저장소 상태(커밋·파일)"를 기준으로 이어서 작업한다는 점이다. 그래서 다음 섹션의 CLAUDE.md 작업 절차가 더 중요해진다.

---

## CLAUDE.md에 작업 시작/종료 절차 추가

여러 컴퓨터를 오가며 작업하다 보니, 매번 "지난번에 어디까지 했는지", "다음에 뭘 해야 하는지"를 새 세션에서 다시 설명해야 하는 비효율이 생겼다. 이를 CLAUDE.md에 명시해서 해결했다.

```markdown
## 현재 작업 브랜치
- 브랜치명: feature/build
- 작업 내용: 대시보드 차트 기능 구현
- 시작일: 2026-05-30

## 작업 시작 전 체크리스트
1. main 브랜치로 전환 후 pull
2. feature/build 브랜치로 전환
3. main 최신 내용 반영 (rebase)
4. 지난 작업은 build-docs/work-log.md 확인
5. 이어서 해야 할 내용은 build-docs/next-tasks.md 확인

## 작업 종료 절차
1. 변경사항 전체 확인 (git status)
2. 작업 내용 커밋 (커밋 메시지는 작업 내용 간단히)
3. feature/build 브랜치에 push
4. build-docs/work-log.md에 오늘 작업 내용 기록
5. 다음 작업 시 이어할 내용을 build-docs/next-tasks.md에 추가
```

이렇게 설정해두면, 새 컴퓨터에서는 단 한 마디로 작업을 이어갈 수 있다.

- 시작할 때: **"CLAUDE.md 작업 시작 절차 따라서 준비해줘"**
- 종료할 때: **"CLAUDE.md 작업 종료 절차 따라서 마무리해줘"**

이 한 마디로 Claude Code가 CLAUDE.md를 읽고 main 전환 → pull → feature/build 전환 → rebase 반영 → next-tasks.md 요약까지 자동으로 처리해준다.
