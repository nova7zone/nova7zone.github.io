3. Claude Code 질의

----------------------------------

진단 결과

원인: screening/market_filter.py의 is_market_buyable()이 KOSPI 지수(0001)의 일봉 데이터를 api_client.get_daily_candles()로 조회했는데 0건이 반환되어, "데이터 부족 → 매수 차단"으로 빠진 것입니다. 매도할 보유 종목도 없었으니 결국 이 한 번의 차단 때문에 장중 매매가 전혀 일어나지 않았습니다.

왜 0건이 나왔는가 — get_daily_candles()(api/kis_api.py:319-337)는 종목 코드 종류와 무관하게 항상 FID_COND_MRKT_DIV_CODE: "J"(주식 시장 구분 코드)로 KIS의 개별종목용 일봉 엔드포인트(inquire-daily-itemchartprice, TR FHKST03010100)를 호출합니다.

문제는 market_filter.py:94와 market_regime.py:195가 이 함수에 종목 코드가 아닌 KOSPI 지수 코드("0001") 를 넘긴다는 점입니다. 지수의 일봉 시계열은 개별종목 엔드포인트로 조회되지 않고, 별도의 지수용 엔드포인트(inquire-daily-indexchartprice, TR FHKUP03500100)를 FID_COND_MRKT_DIV_CODE: "U"로 호출해야 합니다 — 바로 옆의 get_index_price()가 현재가 조회 시 정확히 "U"를 쓰고 있는 것과 대조됩니다.

즉 "J" + 개별종목 엔드포인트로 지수 코드를 조회하니 KIS 측에서 빈 output2(또는 실패)를 반환했고, len(candles) == 0 → "KOSPI 일봉 데이터 부족 (0/20일)" → 보수적으로 매수 차단된 것입니다.

왜 지금까지 발견되지 않았는가:

일회성 네트워크 문제가 아니라 구조적 버그입니다 — KOSPI 지수 코드로 get_daily_candles()를 호출하는 한 항상 0건이 돌아옵니다.

screening/market_regime.py:195도 같은 함수를 같은 방식으로 호출하지만, 거기서는 not candles 시 그냥 "chop"으로 폴백해 조용히 넘어가므로 증상이 드러나지 않습니다.

tests/test_filters.py는 get_daily_candles를 MagicMock으로 대체해 호출 파라미터와 무관하게 캔들 리스트를 그대로 반환하므로, 이 버그는 실제 KIS API와 통신할 때만 드러납니다.

수정 방향(제안): api_constants.py에 지수 일봉 엔드포인트 경로와 TR ID(FHKUP03500100)를 등록하고, kis_api.py에 FID_COND_MRKT_DIV_CODE: "U"를 사용하는 별도의 get_index_daily_candles() 같은 메서드를 추가한 뒤, market_filter.py/market_regime.py가 KOSPI 지수 조회 시 그 메서드를 쓰도록 변경하는 것입니다.

원하시면 trading/screening 변경 규칙(5단계: clarify→design→plan→code→verify)에 따라 이 수정을 진행하겠습니다 — 진행할까요?

4. 작업 완료

--------------------------------------------------​

수정 완료

KOSPI 지수 일봉 조회가 항상 0건을 반환해 매수가 차단되던 구조적 버그를 수정했습니다.

변경 내역:

api_constants.py — 업종/지수 전용 일봉 엔드포인트 경로(INDEX_DAILY_CANDLE_PATH)와 TR ID(FHKUP03500100) 상수 추가

api/kis_api.py — get_index_daily_candles() 신규 메서드 추가. FID_COND_MRKT_DIV_CODE: "U"로 업종 전용 엔드포인트(inquire-daily-indexchartprice)를 호출하고, 날짜 범위(FID_INPUT_DATE_1/FID_INPUT_DATE_2)를 계산해 전달하도록 구현 (KIS 공식 예제·백테스터 코드로 엔드포인트·TR ID·필드명을 교차 검증함)

api/dry_run_client.py — 동일 시그니처 mock 메서드 추가

screening/market_filter.py — get_daily_candles() → get_index_daily_candles()로 교체하고, 종가 필드를 stck_clpr(개별종목용) → bstp_nmix_prpr(지수용)로 수정

screening/market_regime.py — 동일하게 get_index_daily_candles() 호출로 교체 (_extract_closes는 이미 두 필드명 모두 처리하도록 되어 있어 변경 불필요)

tests/conftest.py — make_index_candles() 헬퍼 추가, mock_api 픽스처에 get_index_daily_candles mock 등록

tests/test_filters.py — is_market_buyable()에 대한 직접 테스트 5건 추가 (지수 전용 엔드포인트 호출 검증, 빈 응답 시 차단, bstp_nmix_prpr 필드 파싱, MA 하회 시 차단 등) — 기존엔 이 함수에 대한 단위 테스트가 전혀 없어서 이 버그가 그동안 발견되지 않았던 점을 보강

pytest tests/ 164개 전부 통과 확인했습니다. 이제 다음 장중 실행부터는 KOSPI 지수 일봉이 정상 조회되어 시장 필터가 실제 이동평균 기준으로 매수 허용/차단을 판단할 것입니다.
