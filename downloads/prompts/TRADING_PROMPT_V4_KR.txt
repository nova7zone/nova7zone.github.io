SECTION 1. 핵심 목표
════════════════════════════════════════════════════════════════

- 한국투자증권(KIS) REST API 기반 한국 주식 자동매매 시스템
- 크론(Cron) 기반 매 15분마다 실행되며, 각 실행 완료 후 즉시 종료 (상태 비저장, stateless)
- 모드 완벽 분리: 실투자(live) / 모의투자(paper) / 백테스트(backtest)
- 조건 기반 점수화(Scoring) + 프리셋 전략 시스템
- Oracle Cloud Ubuntu 22.04 서버에서 장기간 무인 작동하도록 설계됨
- 텔레그램 실시간 알림 + 투자 일지 자동 업데이트
- 데몬(Daemon) 없음. 무한 루프 없음. 순수 크론 기반 단발성(single-shot) 실행 모델.
- 비동기 주문 관리 (다음 크론 실행 시 미체결 주문 확인).

SECTION 2. 실행 환경 (Oracle Cloud Ubuntu 22.04)
════════════════════════════════════════════════════════════════

2-1. 서버 사양
  - 클라우드   : Oracle Cloud Infrastructure (OCI) Always Free Tier
  - OS         : Ubuntu 22.04 LTS (64-bit)
  - 사양       : 1 OCPU (AMD), 1 GB RAM, 47 GB Block Volume
  - 파이썬     : 3.11 (아직 설치되지 않음 → INSTALL.md에 설정 안내 작성)
  - 접속       : SSH 키 기반 (ubuntu 계정)
  - 실행 계정  : ubuntu (sudo 권한 보유)

2-2. 시스템 타임존 (KST 필수)
  - 한국 시장 시간은 KST 기준이므로 서버는 반드시 Asia/Seoul로 설정되어야 함:
      sudo timedatectl set-timezone Asia/Seoul
      timedatectl   # 확인: Time zone: Asia/Seoul (KST, +0900)

2-3. 방화벽 (UFW — firewalld 아님)
  - Ubuntu 22.04는 UFW를 사용함:
      sudo ufw allow out 443/tcp
      sudo ufw allow out 9443/tcp
      sudo ufw allow out 29443/tcp
  - OCI 콘솔 Security List의 Egress 규칙도 확인할 것

2-4. 파이썬 설치 절차 (Ubuntu 22.04, INSTALL.md에 포함)
  Step 1: sudo apt update && sudo apt upgrade -y
  Step 2: sudo apt install -y python3.11 python3.11-venv python3.11-dev python3-pip git curl wget nano build-essential
  Step 3: sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.11 1
  Step 4: mkdir -p ~/trader && cd ~/trader
          python3 -m venv venv
          source venv/bin/activate
  Step 5: pip install -r requirements.txt

2-5. 크론탭(Crontab) 설정 (Ubuntu 22.04, 경로: /home/ubuntu/trader)
  - cron 활성화:
      sudo systemctl enable cron && sudo systemctl start cron
  - crontab -e 에 다음 추가:

    # 장중 매 15분마다 실행 (월-금 09:00-15:30 KST)
    */15 9-15 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 16:00 일일 보고서 생성
    0 16 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --report-daily >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 17:00 월간 보고서 확인 (runner.py 내부에서 영업일 기준 말일인지 판별)
    0 17 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --check-monthly >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 15:35 투자 일지 업데이트 (장 마감 종가 반영)
    35 15 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --update-journal >> /home/ubuntu/trader/logs/cron.log 2>&1

SECTION 6. KIS REST API 명세
════════════════════════════════════════════════════════════════

6-1. api/token_manager.py
  - 한국투자증권(KIS) 요구사항에 맞는 OAuth2 자동 갱신 로직

  - filelock을 통한 경쟁 조건(Race condition) 방지:
      with FileLock("data/token_cache.lock"):

  - API 통신 로직만 포함

      get_price_info(code) → dict
      get_daily_candles(code, count) → list
      get_stock_list(market, sort, count) → list
      get_balance() → dict
      get_today_trades() → list
      send_buy_order(code, qty, price, order_type) → dict
      send_sell_order(code, qty, price, order_type) → dict
      cancel_order(order_no, code, qty) → bool
      check_order_status(order_no) → dict
      log_api_stats() → None
  - API 호출 카운터: 실행당 총 호출 수 로그 기록
  - 재시도 로직: 최대 3회 시도, 1초 간격


════════════════════════════════════════════════════════════════
SECTION 7. 중복 주문 방지 + 비동기 체결 확인
════════════════════════════════════════════════════════════════

7-1. 프로세스 락(Process Lock)
  - runner.py가 시작 시 filelock을 통해 data/runner.lock 획득
  - 타임아웃: 1초 — 이미 잠겨 있다면 즉시 sys.exit(0)

7-2. 스레드 세이프 JSON I/O (utils/file_utils.py)
  - 모든 JSON 파일 읽기/쓰기는 filelock으로 보호됨
  - 함수: load_json_locked(path), save_json_locked(path, data)

7-3. 주문 전 잔고 확인 (trading/buy.py)
  - 각 종목 주문 전 잔고를 다시 조회함 (이전 매수 내역 반영)
  - 조건: 가용 자산 >= 수량 × 가격 + 최소_가용_버퍼(min_available_buffer)

7-4. 비동기 체결 확인 (trading/order_manager.py)
  - 절대 대기(poll)하거나 sleep 하지 마세요. 실행 스레드를 블로킹하면 안 됩니다.
  - 주문 실행 후, 주문 세부 정보를 data/pending_orders.json 에 저장합니다.
  - 다음 15분 크론 실행 시, runner.py는 가장 먼저 `order_manager.check_pending_orders()`를 호출해야 합니다.
  - 대기 중인 주문이 체결(FILLED)된 경우: trade_log 업데이트, holdings.json 업데이트, 텔레그램 알림 전송.
  - 15분 후에도 대기 중인 주문이 미체결(UNFILLED)된 경우: API를 통해 주문 취소, pending_orders에서 제거, 텔레그램 경고 전송.

SECTION 9. 매수 로직 (trading/buy.py)
════════════════════════════════════════════════════════════════

9-1. 실행 단계 (비동기)
  1) 시장 상황 필터 확인 → 차단 시 종료(return)
  2) 스크리닝 실행 → 정렬된 후보 목록 도출 (점수 높은 순)
  3) 계좌 잔고 조회
  4) 현재 보유 종목 수 확인 → 최대 보유 종목 수 도달 시 종료
  5) 오늘 이미 매수한 종목 제외 (trade_log 확인)
  6) 현재 보유 중인 종목 제외 (holdings.json 확인)
  7) 각 후보 종목에 대해:
     (a) 잔고 재조회 → 가용 자금 확인
     (b) 매수 수량 계산: qty = 종목당_매수_금액 // 현재가
     (c) 주문 실행 (send_buy_order)
     (d) 다음 실행 시 비동기 체결 확인을 위해 pending_orders.json에 주문 기록.


════════════════════════════════════════════════════════════════
SECTION 10. 매도 로직 (trading/sell.py)
════════════════════════════════════════════════════════════════

10-1. 매도 우선순위 (순서대로 평가됨)
  우선순위 1: 손절 (수익률 <= 손절_비율)
  우선순위 2: 익절 (수익률 >= 익절_비율)
