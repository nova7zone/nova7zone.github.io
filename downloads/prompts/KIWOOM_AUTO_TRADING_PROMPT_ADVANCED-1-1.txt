KIWOOM REST API 기반 자동매매 시스템 – 초고급 통합 프롬프트
================================================

[목표]
- 키움증권 REST API를 이용한 실전 자동매매 시스템을 Python으로 구현한다.
- 단기 실행(crontab 5분 주기) 기반 구조를 유지한다.
- 실매매/백테스트/리포트/조건 프리셋/점수 기반 의사결정을 모두 포함한다.
- 초보 서버(Ubuntu 22.04)에서도 설치 및 운영 가능해야 한다.

────────────────────────────────────────
1. 코드 품질·테스트 강제 요구사항
────────────────────────────────────────
- 모든 Python 파일은 다음을 반드시 포함한다.
  * 모듈 docstring
  * 함수별 상세 주석
  * type hint
- 공통 규칙
  * 전역 변수 사용 금지
  * 설정값은 settings/*.yaml 에서만 관리
  * 하드코딩 금지
- 테스트
  * core 로직은 unittest 또는 pytest 기준 테스트 코드 작성
  * 실제 주문 API는 mock 처리
  * tests/ 폴더 분리

────────────────────────────────────────
2. 키움 REST API 실명세 반영 요구
────────────────────────────────────────
- 인증
  * OAuth2 기반 access token 발급
  * 토큰 만료 시 자동 재발급
- 주문
  * 시장가 / 지정가 선택 가능
  * 매수/매도 함수 완전 분리
- 조회
  * 계좌 잔고
  * 체결 내역
  * 일/월 누적 손익
- API 에러 처리
  * 재시도 로직 (최대 3회, 지수 백오프)
  * 로그 기록 (logs/)

────────────────────────────────────────
3. 백테스트 + 실매매 분리 구조
────────────────────────────────────────
- 실행 모드
  * mode: live | paper | backtest
- 동일한 전략 코드를 사용
- 데이터 소스만 분리
  * live: 키움 REST
  * backtest: CSV / DB
- 백테스트 결과
  * 월별 수익률
  * MDD (Maximum Drawdown)
  * 승률
  * 거래 횟수

────────────────────────────────────────
4. 조건·점수·프리셋 시스템
────────────────────────────────────────
- 1차 스크리닝
  * 유동성
  * 거래대금
  * 기본 필터
- 2차 스크리닝
  * 조건 파일 개별 관리 (conditions/*.py)
  * 이동평균 교차 (MA cross)
  * 거래량 급증 (Volume spike)
  * 볼린저 밴드 (Bollinger Band)
  * 스톡캐스틱 (Stochastic)
- 점수 시스템
  * 조건별 가중치
  * 종합 점수 산출
- 프리셋
  * presets/*.yaml
  * conservative (점수 threshold: 25)
  * neutral (점수 threshold: 20)
  * aggressive (점수 threshold: 15)
  * presets에서 활성 조건 + 가중치 정의
- runner.py는 설정만 로드, 조건 로직 직접 수정 금지

────────────────────────────────────────
5. 리포트 & 투자일지 & 텔레그램
────────────────────────────────────────
- 장 종료 리포트
  * 일별 손익
  * 체결 요약
  * 보유 종목 현황
- 월말 리포트
  * 월 수익률
  * 누적 수익률
  * 전략별 성과
- 투자일지 (CSV 자동 기록)
  * 매수 기록: 종목, 수량, 매수가, 조건, 초기 투자비용
  * 매도 기록: 종목, 수량, 매도가, 보유기간, 수익률, 매도이유
- 텔레그램
  * 봇 토큰/채널 ID는 settings/secret.yaml
  * 메시지 자동 전송
  * 실패해도 거래 계속 진행

────────────────────────────────────────
6. 실행 구조
────────────────────────────────────────
- crontab 설정
  */5 * * * * /usr/bin/python3 /path/to/runner.py
- runner.py 동작
  * 설정 로드
  * 장중 여부 판단
  * 매수/매도 판단
  * 주문 후 종료 (stateless)
  * 에러 발생 시 로그 + 종료
