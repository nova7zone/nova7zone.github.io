SECTION 2. EXECUTION ENVIRONMENT (Oracle Cloud Ubuntu 22.04)
════════════════════════════════════════════════════════════════

2-1. Server Specifications
  - Cloud      : Oracle Cloud Infrastructure (OCI) Always Free Tier
  - OS         : Ubuntu 22.04 LTS (64-bit)
  - Specs      : 1 OCPU (AMD), 1 GB RAM, 47 GB Block Volume
  - Python     : 3.11 (not yet installed → setup in INSTALL.md)
  - Access     : SSH key-based (ubuntu account)
  - Run user   : ubuntu (with sudo privileges)

2-2. System Timezone (KST mandatory)
  - Korean market hours are KST-based — server MUST be set to Asia/Seoul:
      sudo timedatectl set-timezone Asia/Seoul
      timedatectl   # verify: Time zone: Asia/Seoul (KST, +0900)

2-3. Firewall (UFW — NOT firewalld)
  - Ubuntu 22.04 uses UFW:
      sudo ufw allow out 443/tcp
      sudo ufw allow out 9443/tcp
      sudo ufw allow out 29443/tcp
  - Also check OCI Console Security List Egress rules

2-4. Python Installation Procedure (Ubuntu 22.04, in INSTALL.md)
  Step 1: sudo apt update && sudo apt upgrade -y
  Step 2: sudo apt install -y python3.11 python3.11-venv python3.11-dev python3-pip git curl wget nano build-essential
  Step 3: sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.11 1
  Step 4: mkdir -p ~/trader && cd ~/trader
          python3 -m venv venv
          source venv/bin/activate
  Step 5: pip install -r requirements.txt

2-5. Crontab Setup (Ubuntu 22.04, path: /home/ubuntu/trader)
  - Enable cron:
      sudo systemctl enable cron && sudo systemctl start cron
  - Add to crontab -e:

    # Every 15 minutes during market hours (Mon-Fri 09:00-15:30 KST)
    */15 9-15 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 16:00 Daily report
    0 16 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --report-daily >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 17:00 Monthly report check (runner.py determines if last biz day)
    0 17 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --check-monthly >> /home/ubuntu/trader/logs/cron.log 2>&1

    # 15:35 Investment journal update (post-market closing price)
    35 15 * * 1-5 /home/ubuntu/trader/venv/bin/python3 /home/ubuntu/trader/runner.py --update-journal >> /home/ubuntu/trader/logs/cron.log 2>&1

════════════════════════════════════════════════════════════════

7-1. Process Lock
  - runner.py acquires data/runner.lock via filelock at startup
  - Timeout: 1 second — if already locked, sys.exit(0) immediately

7-2. Thread-Safe JSON I/O (utils/file_utils.py)
  - All JSON file reads/writes protected by filelock
  - Functions: load_json_locked(path), save_json_locked(path, data)

7-3. Pre-Order Balance Check (trading/buy.py)
  - Re-query balance before EACH stock order (reflects prior buys)
  - Condition: available >= qty × price + min_available_buffer

7-4. Asynchronous Fill Confirmation (trading/order_manager.py)
  - DO NOT poll or sleep. NO blocking the execution thread.
  - After placing an order, save order details to data/pending_orders.json.
  - On the NEXT 15-minute cron execution, runner.py must FIRST call `order_manager.check_pending_orders()`.
  - If a pending order is FILLED: update trade_log, update holdings.json, and send telegram alert.
  - If a pending order is UNFILLED after 15 minutes: cancel the order via API, remove from pending_orders, and send telegram warning.

SECTION 9. BUY LOGIC (trading/buy.py)
════════════════════════════════════════════════════════════════

9-1. Execution Steps (Asynchronous)
  1) Check market regime filter → return if blocked
  2) Run screening → sorted candidate list (highest score first)
  3) Query account balance
  4) Check current holding count → return if at max_holding_count
  5) Exclude stocks already bought today (trade_log check)
  6) Exclude currently held stocks (holdings.json check)
  7) For each candidate stock:
     (a) Re-query balance → check available funds
     (b) Calculate qty: qty = buy_amount_per_stock // price
     (c) Execute order (send_buy_order)
     (d) Record order to pending_orders.json for async fill checking on next run.
